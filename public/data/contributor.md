# Contributing to MAGIC

Thanks for helping improve the framework. Contributions from real operators are what make this useful.

---

## What you're doing

You're proposing a change to the **MAGIC Controls Framework** — the open source control set that powers this tool.

Your change will be submitted as a GitHub Issue. A maintainer will review it and open a Pull Request if it's accepted. Once merged, the updated controls go live for everyone.

---

## What makes a good contribution

Before submitting, ask yourself:

- **Is it operational?** The control must describe something that can be done, configured, or evidenced — not a principle or aspiration.
- **Does it have an owner?** Every control needs a real role attached to it (vCIO, security lead, department owner, etc.).
- **Can it produce evidence?** If there's no way to prove it happened, it doesn't belong.
- **Does it scale?** It should work across customers of different sizes and industries, not just one scenario.
- **Does it fit GSO?** If it doesn't clearly support Govern, Secure, or Operate, it's probably out of scope.

---

## What's out of scope

Please don't submit:

- Tool recommendations or vendor comparisons
- Prompt tips or productivity hacks
- Ethics statements that can't be enforced
- One-time assessments with no ongoing cadence

---

## After you submit

- A GitHub Issue is created automatically with your proposed changes.
- Maintainers review for framework fit, consistency, and quality.
- If accepted, a Pull Request is opened and merged — your handle is added to the contributors list.
- If declined, the issue will be closed with a note explaining why.

---

## You're adding your name to this

When your contribution is accepted, your GitHub handle is recorded in the contributors list as part of the framework's history.

Contribute something real.
